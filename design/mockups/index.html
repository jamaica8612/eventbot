Redesign Mockups · 2026-05-18

EventBot 전체 리디자인 시안 6종

"택배일 중 한 손 딸깍" 초기 컨셉을 벗고 본격 이벤트 관리 도구로 진화하는 방향을 6가지 패러다임으로 풀어봤다. 각 시안을 클릭하면 새 탭에서 열린다. 하나를 선정하면 후속 PR에서 실제 React 컴포넌트로 이식한다.

01📊

대시보드 중심

PC 우선통계 친화3컬럼

좌측 네비 + 상단 KPI(누적상금/당첨률/오늘마감/미수령) + 차트 + 일정 사이드바.

강점 한눈에 현황 파악, 데이터 의사결정 · 약점 행동(응모) 자체는 보조적
02📥

인박스 트리아지

Gmail/Linear 스타일키보드3패널

좌측 폴더 + 중앙 컴팩트 리스트 + 우측 디테일. J/K 이동, E/L 액션. 처리 속도 최우선.

강점 빠른 처리·컨텍스트 유지 · 약점 초보 사용자에게 단축키 학습 비용
03🗂

칸반 보드

라이트DnD워크플로우

응모대기 → 임시저장 → 참여완료 → 당첨 → 미당첨 컬럼. 드래그앤드롭으로 상태 변경.

강점 단계 직관, 진행감 · 약점 모바일 약함, 항목 많아지면 세로 스크롤 부담
04📰

매거진 / 카드 그리드

라이트메이슨리이미지 강조

Pinterest 스타일 비대칭 그리드. 플랫폼 아이콘과 경품 이미지로 시각적 매력 극대화.

강점 발견·브라우징, 첫인상 좋음 · 약점 정보 밀도 낮고 데이터 작업에 약함
05📋

밀도형 데이터 테이블

라이트Airtable인라인 편집

정렬·필터·그루핑 강력. 한 화면 25행+, 셀 인라인 편집. 분석·정리 흐름 최적.

강점 정보 밀도 최강, 일괄 작업 · 약점 시각적 매력↓, 모바일 호환 어려움
06📱

모바일 모던

다크바텀시트3화면

Toss/Linear 모바일 스타일. 홈 + 바텀시트 디테일 + 수령함 3개 화면. 큰 터치 타겟, 스와이프 액션.

강점 모바일 일등급, 익숙한 UX · 약점 PC 활용도 낮음, 정보 밀도 중하

비교 포인트

  • 주 사용 환경이 PC면 01·05, 데스크탑+모바일 균형은 02·03, 모바일 우선은 06, 발견 중심은 04
  • 핵심 사용자 행동이 빠른 처리 → 02, 진행 추적 → 03, 분석 → 05, 발견 → 04
  • 구현 비용 (대략): 04 ≤ 06 ≤ 01 < 03 < 02 < 05 (DnD/스프레드시트형이 비싸다)

각 HTML은 self-contained — 외부 의존성은 Pretendard CDN과 picsum 이미지뿐. 시안 선정 후 토큰/레이아웃을 styles.css 베이스에 통합한다.
