# EventBot 리디자인 시안 6종 — 비교 메모

각 시안은 `design/mockups/0X-*.html` 에 self-contained HTML로 있다. 진입점은 [`index.html`](../index.html).

## 한눈에 비교

| # | 시안 | 패러다임 | 테마 | 강점 | 약점 | 구현 비용 |
|---|------|---------|------|------|------|----------|
| 01 | 대시보드 | KPI + 차트 + 일정 | 다크 | 현황 파악, 데이터 의사결정 | 행동(응모)이 보조적 | 중 |
| 02 | 인박스 트리아지 | 3패널 + 키보드 단축키 | 다크 | 빠른 처리, 컨텍스트 유지 | 단축키 학습 비용 | 중상 |
| 03 | 칸반 보드 | 컬럼 + 드래그앤드롭 | 라이트 | 단계 직관, 진행감 | 모바일 약함 | 상 |
| 04 | 매거진 | 메이슨리 카드 그리드 | 라이트 | 시각적 매력, 발견 | 정보 밀도 낮음 | 하 |
| 05 | 데이터 테이블 | Airtable 인라인 편집 | 라이트 | 정보 밀도 최강, 일괄 작업 | 모바일 부적합 | 상 |
| 06 | 모바일 모던 | 바텀시트 + 큰 터치 | 다크 | 모바일 일등급 | PC 활용도 ↓ | 중 |

## 선택 가이드

- **PC에서 데이터 보면서 의사결정** → 01 (대시보드) 또는 05 (테이블)
- **응모 처리량이 많아 속도가 핵심** → 02 (인박스)
- **응모 → 결과 단계가 머릿속에 안 잡힘** → 03 (칸반)
- **사용자에게 "즐거운" 발견을 주고 싶음** → 04 (매거진)
- **현재 모바일 사용자 비중이 60%+** → 06 (모바일 모던)

## 공통 데이터

[`seed-data.js`](./seed-data.js) — 8건의 더미 이벤트 (필드: `title`, `platform`, `prizeText`, `prizeAmount`, `totalWinnerCount`, `deadlineText`, `status`, `resultStatus`, `receiptStatus` 등). 각 HTML은 인라인으로 동일 데이터를 사용한다.
