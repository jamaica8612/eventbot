# 당첨노트 — 디자인 규칙 (Design Rules)

> **이 문서는 "강제 규칙"입니다.** 디자인을 만들거나 고칠 때 여기 적힌 걸 어기지 마세요.
> 새 화면·컴포넌트를 추가할 때도 **새 값을 발명하지 말고** 아래 토큰·패턴만 조합하세요.
> 규칙을 바꿔야 한다면, 즉흥 수정 대신 **이 문서를 먼저 고치고** 전체에 일괄 반영합니다.

---

## 0. 황금률 (절대 어기지 말 것)

1. **색은 토큰만 쓴다.** `var(--accent)` 등 `styles.css`에 정의된 변수만. 새 hex를 코드에 직접 박지 않는다. 꼭 필요하면 토큰을 추가하고 라이트/다크 둘 다 정의한다.
2. **라이트가 기본, 다크는 동등 시민.** 무엇을 만들든 `[data-theme="dark"]`에서 깨지지 않아야 한다. 흰색/검정을 직접 쓰지 말고 `--surface`/`--text`를 쓴다.
3. **강조색은 "강조"에만.** `--accent`(인디고)는 주요 액션·활성 상태·브랜드에만. 정보 위계를 위한 색이지 장식이 아니다.
4. **주의 위계가 최우선.** 오늘 발표 / 발표일 지남 / 미수령 / 오늘 마감 → 이 4개는 **항상 가장 눈에 띄어야** 한다(§7).
5. **숫자엔 `tabular-nums`, 한글엔 `word-break: keep-all`.** 예외 없음.
6. **터치 타깃 ≥ 44px(모바일).** 본문 텍스트 ≥ 14px.
7. **장식 금지.** 의미 없는 그라데이션·이모지·통계·아이콘을 넣지 않는다(§11).

## 1. 컬러 토큰 (단일 소스: `styles.css`)

| 용도 | 토큰 | Light | Dark |
|---|---|---|---|
| 앱 배경 | `--bg` | #eef0f4 | #0c0e13 |
| 카드/패널 | `--surface` | #ffffff | #161922 |
| 살짝 가라앉은 면 | `--surface-2` | #f5f6f9 | #1c202b |
| 더 가라앉은 면/배지 | `--surface-3` | #eceef3 | #232834 |
| 기본 테두리 | `--border` | #e3e6ec | #2a3040 |
| 강한 테두리/입력 | `--border-strong` | #d3d8e0 | #39414f |
| 본문 텍스트 | `--text` | #1b1e27 | #e9ebf1 |
| 보조 텍스트 | `--text-2` | #565d6d | #a4abbb |
| 흐린 텍스트/메타 | `--text-3` | #8a91a1 | #6f7787 |

### 강조 / 의미색 — **반드시 용도대로만**
| 의미 | 채움색 | 약한 배경 | 글자색 | **언제 쓰나** |
|---|---|---|---|---|
| 강조(brand) | `--accent` #4f46e5 | `--accent-weak` | `--accent-text` | primary 버튼, 활성 칩/탭, 링크, 포커스 |
| 주의(warn) 🟠 | `--warn` #d97706 | `--warn-weak` | `--warn-text` | **오늘 발표 · 미수령 · 내일/임박 마감** |
| 시급(urgent) 🔴 | `--urgent` #e11d48 | `--urgent-weak` | `--urgent-text` | **발표일 지남 · 오늘 마감 · 만료 임박** |
| 당첨(win) 🟢 | `--win` #0f9d58 | `--win-weak` | `--win-text` | 당첨 결과, 참여완료, 수령완료, 성공 토스트 |
| 미당첨(lose) ⚪ | `--lose` #8a91a1 | `--lose-weak` | — | 미당첨(중립 처리, 빨강 쓰지 말 것) |
| 정보(info) 🔵 | `--info` #2563eb | `--info-weak` | — | 동기화 진행, 예정 발표(N일 후) |

**규칙**
- 미당첨은 **부정/빨강이 아니라 중립 회색**(`--lose`). 빨강(urgent)은 "지금 행동 필요"에만.
- 한 화면에서 urgent는 진짜 시급한 것에만. 남발하면 위계가 무너진다.
- "당첨"의 초록과 "참여완료" 초록은 같은 `--win` 계열로 통일.

### 플랫폼 색 (고정)
YouTube `--yt` #ff0033 · 네이버 `--naver` #03c75a · 슈퍼투데이 `--home` #6d5ef0. 배지 배경은 각 `*-weak`. **플랫폼 색은 플랫폼 식별에만**(다른 의미로 전용 금지).

## 2. 타이포그래피

- **폰트: Pretendard 단일.** 다른 한글 폰트·세리프 섞지 않는다. (영문도 Pretendard로.)
- **금지 폰트:** Inter / Roboto / Arial 등 슬롭 폰트로 바꾸지 말 것.
- 스케일(권장):

| 역할 | px | weight |
|---|---|---|
| 페이지 제목(상단바) | 19 | 800 |
| 섹션/카드 제목 | 15–16 | 700–750 |
| 본문 | 14 | 400–550 |
| 보조/메타 | 12–12.5 | 600 |
| 배지/칩 | 11.5–13 | 650–700 |
| 큰 숫자(요약 스트립) | 23 | 800 |

- **줄바꿈:** `word-break: keep-all`(어절 단위). 배지·버튼·칩·짧은 라벨은 `white-space: nowrap`.
- **숫자:** 통계·금액·카운트·날짜에 `font-variant-numeric: tabular-nums`(`.tnum`).
- letter-spacing: 본문 -0.01em, 큰 제목 -0.02em.

## 3. 모양 (radius / shadow / spacing)

- **Radius 토큰만:** `--r-xs`6 / `--r-sm`9 / `--r-md`13 / `--r-lg`18 / `--r-xl`24. 배지·칩·토글은 99px(pill).
  - 입력/작은 버튼=`--r-sm`, 카드/패널=`--r-md`, 시트/모달=`--r-lg`.
- **Shadow 토큰만:** `--shadow-1`(카드 기본) / `--shadow-2`(hover·떠있는 요소) / `--shadow-3`(모달·드로어). 임의 box-shadow 금지.
- **스페이싱:** 4의 배수 기준(8·12·16…). 카드 내부 padding 14–16. 요소 묶음은 **flex/grid + `gap`**으로(인라인 여백·개별 margin 나열 금지).

---

## 4. 컴포넌트 계약 (이 스펙대로만)

> 모든 공통 컴포넌트는 `components.jsx`에 있다. **새로 만들지 말고 재사용**. 변형이 필요하면 prop을 추가.

- **Badge** `tone`(muted/accent/warn/urgent/win/lose/info), pill, 11.5px/650, `nowrap`. 상태·메타 라벨 전용.
- **Chip** 필터/정렬용 pill 버튼. 활성=의미색 채움+흰 글자, 비활성=`--surface`+1px `--border`. 카운트는 우측 작은 배지.
- **Btn** `variant`: `primary`(강조 액션·화면당 1~2개) / `default`(중립) / `soft` / `win` / `urgent` / `warn` / `ghost`(저강조) / `outline`. radius `--r-sm`, weight 650.
  - **primary는 화면의 주요 행동에만.** 모든 버튼을 primary로 칠하지 않는다.
- **SegToggle** 상호배타 상태 전환(결과: 당첨/미당첨/미확인, 수령: 미수령/요청/완료). 활성 칸만 의미색 채움.
- **Switch** on/off 단일 토글(테마·체크박스형 옵션).
- **Card(article)** `--surface` + 1px `--border` + `--r-md` + `--shadow-1`, hover `--shadow-2`. 비활성/제외는 opacity 0.62.
- **입력(input/textarea)** `inputStyle` 사용: `--surface` 배경, 1px `--border-strong`, `--r-sm`, padding 8–11. textarea는 IME 안전(조합 중 값 변형 금지, blur 커밋).
- **Overlay/Sheet** 모바일=바텀시트(slideUp, 상단 radius `--r-lg`), 데스크톱=중앙 모달 또는 우측 드로어. backdrop `rgba(15,18,30,.45)`.

**do / don't**
- ✅ 같은 의미엔 같은 컴포넌트·같은 톤.
- ❌ 비슷한 걸 매번 인라인으로 새로 그리기. ❌ 버튼/배지에 임의 색.

---

## 5. 레이아웃 & 반응형

- **분기점 900px** 하나로 통일. 데스크톱: 좌측 사이드바 `--nav-w`(252px) + 본문(max `--maxw` 1180, 중앙). 모바일: 하단 탭바(고정, `env(safe-area-inset-bottom)` 고려) + 상단 컴팩트 헤더.
- 콘텐츠는 항상 `content-inner`(max 1180, 좌우 패딩 26/15)에 담는다.
- 표(응모함·관리자)는 좁은 화면에서 **카드/스택으로 재배치**(가로 스크롤은 관리자 표처럼 불가피할 때만, `min-width` + `overflow-x:auto`).
- 칩 줄은 가로 스크롤(`.chip-row`, 스크롤바 숨김).

---

## 6. 내비게이션

- 항목마다 **카운트 배지**(주의가 필요한 수). 0이면 숨김.
- 활성 탭: `--accent-weak` 배경 + `--accent-text`. 모바일 탭바 활성: 아이콘+라벨 `--accent`.
- 상단바는 항상 **제목 + 결과 개수 + [필터설정]**. 이 구성을 바꾸지 않는다.

---

## 7. 주의 위계 (이 앱의 정체성 — 가장 중요)

화면에서 **시선 우선순위**는 항상:

1. 🔴 **시급(urgent)** — 발표일 지남 · 오늘 마감 · 만료 임박: 빨강 배지 + 행 좌측 컬러 보더 + 요약 스트립 강조.
2. 🟠 **주의(warn)** — 오늘 발표 · 미수령 · 내일 마감: 주황 배지 + 강조 카드.
3. 🟢 당첨/완료(win), ⚪ 그 외 평시.

규칙:
- 요약 스트립의 "오늘 발표"·"미수령"은 **항상 강조 카드**(배경 weak + 테두리 의미색).
- InboxRow는 주의 상태면 **좌측 3px 컬러 보더**로 한눈에. 평시는 transparent.
- 정렬은 항상 "확인 시급 → 미수령 → 완료 → 미당첨". 이 순서를 깨지 않는다.
- 주의 색을 평범한 항목에 쓰지 않는다(남용 시 위계 붕괴).

## 8. 인터랙션 & 모션

- **낙관적 업데이트:** 변경 즉시 화면 반영 → 저장 → 실패 시 sync notice로 알림(§아래).
- **Sync notice(상단 3색 바):** info(파랑)/success(초록)/warning(주황). 저장 실패="DB 저장 실패. 로컬 보관 후 자동 재시도", 복구="밀린 저장 반영됨". 성공 저장은 짧은 "저장됨"(1.6s).
- 트랜지션: `--ease`/`--ease-out`, 120–260ms. 호버 밝기 ±, active scale .97.
- **진입 애니메이션은 끝 상태가 기본이어야 함.** opacity:0에서 시작하는 애니메이션은 `prefers-reduced-motion` 및 정적 캡처에서 내용이 보이도록 처리. 무한 장식 애니메이션 금지.
- 파괴적 동작(대기로 되돌리기·삭제·전체 리셋)은 `confirm` 게이트.

---

## 9. 카피 & 도메인 용어 (라벨 고정 — 바꾸지 말 것)

- 상태: **대기 / 임시저장 / 참여함 / 제외**
- 결과: **결과 미확인 / 당첨 / 미당첨**
- 수령: **미수령 / 수령요청 / 수령완료**
- 발표: **발표일 지남 / 오늘 발표 / N일 후 발표 / 발표일 미정**
- 마감: **오늘 마감 / 내일 마감 / N일 후 마감 / 마감 M/D / 마감 미확인**
- 톤: 1인용 도구답게 담백·친근한 존댓말("…했어요"). 과장·마케팅 문구 금지.

---

## 10. 접근성

- 모바일 터치 타깃 ≥ 44×44. 아이콘 버튼 최소 32(데스크톱)/40(모바일).
- 색만으로 의미 전달 금지 — 항상 **텍스트 라벨 + 색** 병행(배지에 글자 포함).
- 의미색은 글자 대비를 위해 `*-text` 토큰 사용(weak 배경 위).
- 포커스 가시성 유지, `Esc`로 모달/시트 닫기.

---

## 11. 금지 사항 (슬롭 방지)

- ❌ 의미 없는 그라데이션 배경 (브랜드마크·지갑 배너 같은 의도된 곳만 허용).
- ❌ 이모지 남발 — 🎁🏆🔖 정도의 기능적 사용만. 새 이모지 추가 자제.
- ❌ 둥근 모서리 + 좌측 컬러 보더 + 약한 배경의 "AI 카드" 클리셰를 의미 없이 반복.
- ❌ 불필요한 통계/숫자/아이콘(데이터 슬롭). 모든 요소는 제 역할이 있어야.
- ❌ 토큰 밖의 색·그림자·radius·폰트.
- 원칙: **"적을수록 좋다." 빈 곳은 콘텐츠가 아니라 레이아웃으로 푼다.**

---

## 12. 새 요소 추가 체크리스트

새 화면/컴포넌트/상태를 추가하기 전에:

- [ ] 색이 전부 토큰인가? 라이트/다크 둘 다 확인했나?
- [ ] 기존 컴포넌트(Badge/Chip/Btn/Card/Seg…)로 만들 수 있나? (새로 그리지 않았나)
- [ ] 의미색을 §1·§7 규칙대로 썼나? (미당첨=회색, urgent 남용 아님)
- [ ] radius/shadow/spacing이 토큰·4배수인가?
- [ ] 한글 keep-all / 숫자 tabular-nums 적용했나?
- [ ] 900px 모바일에서 안 깨지나? 터치 타깃 44px?
- [ ] 도메인 라벨(§9)을 정확히 썼나?
- [ ] 주의 위계(§7)를 해치지 않나?
- [ ] 장식/슬롭(§11) 없는가?
