# 이벤트봇 개선 1-4 순차 구현 계획

## Summary

사용자가 말한 개선을 순서대로 진행한다. 먼저 계획 문서를 repo에 남기고, 이후 `크롤링 상태 명확화 -> 카드 정보 정리 -> 유튜브 댓글 저장 -> 오래된/중복 이벤트 정리` 순서로 구현한다. 각 단계는 빌드 확인 후 커밋/푸시해서 단계별 롤백이 가능하게 한다.

## Assumptions

- 구현 순서는 1 -> 2 -> 3 -> 4 순서로 고정한다.
- 오래된 이벤트 정리는 실제 DB 삭제가 아니라 UI 숨김부터 적용한다.
- 댓글 저장은 사용자별 데이터로 저장해서 계정별 내용이 섞이지 않게 한다.
- 각 단계 완료 후 커밋/푸시하고, 문제가 생기면 해당 단계 단위로 롤백 가능하게 한다.
